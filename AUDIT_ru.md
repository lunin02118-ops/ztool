> **до** PR #8. После слияния PR #8 (merge `78e679c`) `ZTool.exe`/`ZTool.dll`
> обновлены — текущий статус и хеши в разделе

## 1. Краткий итог

Репозиторий — это **релизный дистрибутив** (а не исходный код) плагина для SolidWorks
**ZTool** (китайского происхождения). Исходного кода (`.cs/.csproj/.resx`) нет — только
скомпилированные сборки .NET и вспомогательные файлы.

Сборки оказались **упакованы и обфусцированы ConfuserEx Compressor** (строки зашифрованы,
тела методов защищены, реальный код спрятан в сжатом payload). Поэтому простая замена строк
была невозможна. По согласованию с заказчиком (вариант C — патч бинарников, при необходимости
с деобфускацией) выполнена **динамическая распаковка → извлечение строк → перевод → патч →
проверка запуском**.

Результат: **standalone-приложение `ZTool.exe` русифицировано** и проверено запуском (см. раздел 4).

## 2. Что находится в репозитории

### Исполняемые модули и библиотеки (бинарники, без исходников)
- `ZTool.exe` (~4.7 МБ) — главное приложение, .NET, упаковано ConfuserEx Compressor.
- `ZTool.dll` (~380 КБ) — отдельная сборка-надстройка для SolidWorks (свои формы PropertyManager).
- `ZToolARM.dll` (~115 КБ) — нативная библиотека x86-64.
- `ZTool Updater.exe` — .NET-апдейтер.
- Сторонние библиотеки: `NPOI*.dll` (Excel), `itextsharp.dll` (PDF).

### Конфигурация (XML)
- `ZTool.settings` (~43 КБ) — основной конфиг (`CConfigDO`). Имена свойств уже на русском;
  значения-перечисления (`文字`, `等于`, `包含`, …) остаются на китайском, **намеренно не тронуты**
  (используются кодом как ключи — см. раздел 5).

### Ресурсы и шаблоны
- Шаблоны чертежей SolidWorks, Excel-шаблоны BOM, тестовые CAD-модели, макросы SolidWorks.
- `help.CHM` (~3.7 МБ) — справка (китайская). `InstallGuide_CN.pdf` — инструкция по установке.
- Повреждённые (mojibake) дубликаты имён папок/файлов — устранены в PR #11
  (дубль `Initialize.exe` удалён; тест-модели сведены в одну папку `TestModel/`).

## 3. Как сделана русификация (методика)

1. **Подтверждение упаковки.** Ресурс `ZTool.9eAd0SlNKphk.png` — не PNG, а зашифрованный payload
   ConfuserEx Compressor (заголовок `60 19 F2 5C…`). Видимый `ZTool.exe` — загрузчик-«стаб».
2. **Динамическая распаковка.** Запуск `ZTool.exe` → дамп процесса из памяти (pe-sieve) →
   загрузка реальной сборки через dnlib (2476 расшифрованных китайских строк-литералов).
3. **Обход anti-unpack.** Метод `code::HasShell(string)` проверяет, что сборка запущена «из-под»
   упаковщика, и при провале молча завершает процесс. Пропатчен на `return true`
   (нейтрализован только self-check распаковки; **система лицензирования/триала не затрагивалась**).
4. **Извлечение строк** трёх типов:
   - литералы кода (`ldstr`);
   - строки форм WinForms (встроенные `.resources`);
   - подписи ленты/меню — компилированная разметка Windows Ribbon Framework
     (`ZTool.RibbonMarkup.ribbon`, строки в Win32-таблицах `RT_STRING`).
5. **Классификация.** Строки разделены на безопасные к переводу (UI/сообщения) и
   «ключевые» (используются в сравнениях/словарях) — последние не переводились.
6. **Глоссарий 中文→рус** на ~730 UI-строк + 97 подписей ленты.
7. **Патч и пересборка.** Замена `ldstr`, переписывание `.resources`, обновление `RT_STRING`
   в ленте через Win32 `UpdateResource` (с сохранением структуры PE/BML), повторное встраивание
   ленты в сборку. Сборка сохранена через dnlib с `PreserveAll` (идентичность и public key сохранены).

## 4. Что русифицировано (проверено запуском)

- Диалог проверки обновлений: «Найдена новая версия!», «Что нового:», «Сайт загрузки».
- Диалог лицензии: «Действующая лицензия не обнаружена!», кнопки «Проба/Регистрация/Отмена».
- Диалог «Параметры» (FrmOptions): все вкладки (Общие/Эскиз/Материал/Пользовательский список),
  флажки и подписи.
- **Лента целиком**: вкладки «Главная/Спецификация/Упаковка/Инструменты», все кнопки
  (Подключить SW, Сохранить в SW, Экспорт спецификации, Пакетная печать, Замена рамки и стандарта,
  Объединение и разделение PDF, Закрепить столбцы и т.д.) и **меню File**
  (Параметры, Проверить обновления, Регистрация, Справка, О программе, Выход).
- Заголовки таблицы (Номер, Имя файла на диске, Путь) и тексты сообщений (MessageBox).

Итоговый бинарник `ZTool.exe` в репозитории заменён на русифицированную распакованную сборку.

## 5. Что осознанно НЕ переведено (и почему)

- **Надстройка SolidWorks `ZTool.dll`** — это отдельная сборка с собственными формами; её строки
  недоступны статически (ресурсы линкованы/зашифрованы и расшифровываются только внутри
  работающего SolidWorks). Без среды SolidWorks безопасно локализовать и проверить её нельзя.
- **Динамический текст**: список изменений в окне обновления (загружается с сервера) и
  обратный отсчёт триала в заголовке (`试用中...剩余N秒`) — формируются в рантайме.
- **Китайские токены-ключи в `.settings`** (`文字`, `等于`, `包含`, …) — читаются кодом как ключи;
  их перевод сломал бы логику фильтров.
- Часть «ключевых» строк-литералов, участвующих в сравнениях/словарях, оставлена на исходном языке.

## 6. Технические замечания / риски

- Поставляемый `ZTool.exe` — **распакованная** (без ConfuserEx) и пересобранная сборка с
  нейтрализованным self-check распаковки. Это компромисс, выбранный заказчиком (вариант C).
- В некоторых формах с фиксированной шириной элементов более длинный русский текст подрезается
  (косметика; функциональность не нарушена).
- Оригинальный `ZTool.exe` доступен в истории Git (предыдущий коммит) при необходимости отката.

## 7. Рекомендация
Для гарантированно полной и поддерживаемой русификации (включая надстройку SolidWorks)
оптимально получить **исходный код ZTool** и локализовать его штатно (resx/сателлитные сборки).

## Актуализация (PR #8)

Дата: 2026-06-14. PR #8 слит в `main` (merge `78e679c`).

С момента этого аудита по бинарникам выполнены и проверены живыми ретестами
(SolidWorks 2025, сборка `0614-A00.SLDASM`, 8/8 режимов BOM, copy/paste) три фронта:

1. **Шаблоны/экспорт BOM** — Arial, русификация, корректные имена столбцов; 8/8 PASS.
2. **`pmpguard2`** в `ZTool.dll` — снята гонка инициализации
   (модальное окно «Ссылка на объект не указывает на экземпляр…»).
3. **`binderfix`** в `ZTool.exe` — защита небезопасной десериализации:
   `VTBinder` (version-tolerant привязка типов в конфиге) + `SafeListBinder`
   (allow-list при вставке из буфера обмена).

Отдельно по справке: `help.CHM` (китайская) переведена на русский —
**`help_ru.chm`** (см. [`tools/chm-i18n/README.md`](tools/chm-i18n/README.md)).
Это снимает пункт 6 в части «справка китайская», но `ZTool.dll`-надстройка
по-прежнему не локализована (её строки доступны только в среде SolidWorks).
