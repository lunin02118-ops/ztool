  коммита мусора.

  байт-в-байт копия `Initialize.exe`; папка `ІвКФДЈРН/` пересекается с `TestModel/`.

Итоговая оценка по документации: **актуализировать 4 документа, добавить
корневой README+индекс, остальное — точечные правки.**

## 3. Что устарело и почему (доказательно)

1. **Тезис «код/бинарь не изменялся».** `AUDIT_REPORT_RU.md` фиксирует «Код в
   рамках проверки не изменялся». После PR #8 это неверно: пропатчены `ZTool.exe`
   (`binderfix`) и `ZTool.dll` (`pmpguard2`).
2. **Локальные пути и версии.** `AUDIT_REPORT_RU.md` ссылается на
   `D:\Development\ztool\ZTool-original`, `ZTool.exe 3.8.4`, `ZTool.dll 1.8.3.0`
   — это окружение мая-2026, не текущая поставка.
3. **Небезопасная десериализация / PublicKeyToken.** Ни один из 4 корневых
   документов не упоминает `BinaryFormatter`-стоки, `SerializationBinder`,
   `VTBinder`/`SafeListBinder` — то есть раздел безопасности устарел; актуальные
   данные только в `manual-test-reports/` и `BinderInject/README.md`.
4. **Связка для деплоя не зафиксирована в обзорной документации.** Рекомендуемая
   пара (`ZTool.exe binderfix 0BF4CB0B…` + `ZTool.dll pmpguard2 D053542…`) есть
   только в PR-описании, не в README.
