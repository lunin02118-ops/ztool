## 2. Целевая структура: 7 режимов экспорта (RU)

| # | Оригинал (CN)             | Русское имя                                   | type | Эскиз | Фильтр         |
|---|---------------------------|-----------------------------------------------|------|-------|----------------|
| 1 | 汇总BOM                   | Сводная спецификация                          | 0    | Нет   | —              |
| 2 | 汇总BOM（带缩略图）       | Сводная спецификация (с эскизами)             | 0    | Да    | —              |
| 3 | 缩进BOM                   | Иерархическая спецификация                    | 1    | Нет   | —              |
| 4 | 缩进BOM（带缩略图）       | Иерархическая спецификация (с эскизами)       | 1    | Да    | —              |
| 5 | 加工件                    | Обрабатываемые детали                         | 0    | Нет   | ByRuler: «Обрабатываемые детали» |
| 6 | 外购件                    | Покупные изделия                              | 0    | Нет   | ByRuler: «Покупные изделия» |
| 7 | 顶层BOM                   | Верхний уровень                               | 2    | Нет   | —              |

### Семантика типов:
- `type=0` — сводка (flat/summary): все компоненты в одном списке, кол-во суммировано
- `type=1` — иерархия (indented): с вложенностью (уровни)
- `type=2` — только верхний уровень (top-level only)
- `type=3` — только детали (parts only, без подсборок)

### Примечание: тип 3 (только детали)
В оригинале нет отдельного режима type=3. В текущем ZTool.settings он присутствует как «Экспорт сводной спецификации деталей». Предлагаю **оставить** его 8-м опциональным режимом.

## 3. Правила фильтрации (FilterRulesList)

### Формат правила:
```
$ИмяСвойства$     |@#$%|Оператор      |@#$%|Значение
```

### Операторы (внутренние коды ZTool, не переводятся):
- `包含` — содержит (contains)
- `等于` — равно (equals)
- `不等于` — не равно (not equals)

### Русские правила (для production-моделей с русскими свойствами):

| Имя правила                       | type  | Условие                                         |
|-----------------------------------|-------|--------------------------------------------------|
| Обрабатываемые детали             | false | Тип содержит «Мех.обработка;Листовая;Литьё;Сварка» |
| Покупные изделия                  | false | Тип содержит «Покупное;Стандартное»               |
| С обработкой поверхности          | false | Тип содержит «Мех.обработка;Листовая;Литьё;Сварка» И Обработка поверхности ≠ пусто |
| Без обработки поверхности         | false | Тип содержит «Мех.обработка;Листовая;Литьё;Сварка» И Обработка поверхности = пусто  |
| Недостающие обязательные свойства | true  | Тип=пусто ИЛИ Номер чертежа=пусто ИЛИ Материал=пусто ИЛИ Масса=пусто ИЛИ Имя детали=пусто |

> **Важно:** значения в правилах (`Мех.обработка`, `Покупное` и т.д.) должны соответствовать реальным значениям свойства «Тип» в ваших русских моделях. Укажите, какие значения используете — подставлю точные.

## 5. Методика контроля (тестирование всех режимов)

### Pre-flight (автоматизированная проверка конфига):
Расширить `check_bom_template.py`:
1. Проверить наличие всех 7(8) bomsettings в файле
2. Проверить что `bomname` ссылается на существующий шаблон
3. Проверить что FilterRulesList содержит правила, на которые ссылаются режимы с ByRuler=true
4. Проверить что имена правил в RulesList каждого bomsetting есть в FilterRulesList

### Ручное тестирование (на тестовой модели):

| Шаг | Режим                              | Ожидаемый результат                            |
|-----|-------------------------------------|------------------------------------------------|
| 1   | Сводная спецификация                | N строк, все колонки заполнены, без вложенности |
| 2   | Сводная спецификация (с эскизами)   | То же + колонка эскизов с картинками            |
| 3   | Иерархическая спецификация          | Строки с уровнями (Level 1, 2), отступы        |
| 4   | Иерархическая (с эскизами)          | То же + эскизы                                  |
| 5   | Обрабатываемые детали               | Только детали с Тип=Мех.обр./Листовая/...      |
| 6   | Покупные изделия                    | Только детали с Тип=Покупное/Стандартное        |
| 7   | Верхний уровень                     | Только прямые дочерние (Level 1)               |
| 8   | Только детали (опц.)                | Без подсборок, только .SLDPRT                  |

### Критерии PASS:
- Excel-файл создан без ошибок
- Количество строк > 0 (для нефильтрованных) или ≥ 0 (для фильтрованных, если нет таких деталей)
- № п/п заполнен последовательно
- Кол-во заполнено числами > 0
- Свойства заполнены (если propname совпадает с моделью)
- Для эскизных режимов: колонка эскизов содержит изображения
