# Архив старых методик тестирования

Эти документы сохранены только как историческая справка. Для текущих прогонов
после рефакторинга использовать их как основную инструкцию нельзя: часть путей,
хешей, профилей и последовательностей запуска устарела.

Актуальные точки входа:

- `docs/release/FULL_TEST_METHODOLOGY_RU.md` — главный рабочий документ для
  полного функционального прогона и паритета с оригиналом.
- `docs/release/FINAL_ACCEPTANCE_TEST_PLAN_RU.md` — финальный release gate.
- `docs/release/PRODUCTION_READINESS_REPORT_RU.md` — шаблон итогового отчёта.

Если в старой методике есть полезный шаг, его нужно перенести в
`FULL_TEST_METHODOLOGY_RU.md`, а не запускать отдельным параллельным процессом.
