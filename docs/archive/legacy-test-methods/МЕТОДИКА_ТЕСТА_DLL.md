# Методика теста русификации `ZTool.dll` (вкладка/панель в SolidWorks)

Документ для **итеративной** проверки кандидатов `ZTool.dll` (расшифрованных/русифицированных)
в SolidWorks 2025 на твоей машине. Я присылаю сборку DLL → ты прогоняешь по шагам ниже →
заполняешь **шаблон результата** (раздел 7) и присылаешь мне. По твоему отчёту я правлю и шлю
следующую сборку.

> Контекст. `ZTool.dll` защищён коммерческим протектором (шифрование тел методов, искажение
> кучи строк `#US`, перепаковка+шифрование ресурсов, анти-дамп). Любая русифицированная DLL —
> это **изменённый бинарь без защиты**, поэтому единственный надёжный критерий «жива ли сборка» —
> загрузка и работа именно в реальном SolidWorks. Это и есть цель методики.

## 1. Рабочая папка теста (только ASCII-пути!)

Не запускай из путей с не-ASCII символами (китайские папки `TestModel` ломают загрузку моделей).

```
set RUNDIR=D:\ztool-dlltest
mkdir %RUNDIR%
```

Состав `RUNDIR` (как в твоём прошлом успешном прогоне):
- кандидат `ZTool.dll` (который я пришлю);
- `SolidWorksTools.dll` (скопировать из установленного SW — без него RegAsm падает);
- прочие файлы-спутники, если присылаю (`ZTool.settings`, иконки и т.п.).

---

## 2. Бэкап и фиксация версии кандидата

```bat
:: какой именно файл тестируем
certutil -hashfile %RUNDIR%\ZTool.dll SHA256
```
Запиши SHA256 кандидата в отчёт (чтобы не перепутать сборки).

Если раньше регистрировал оригинал — сними регистрацию старого, прежде чем ставить новый
(раздел 6 — откат).

---

## 3. Проверка загружаемости DLL ДО SolidWorks (быстрый фильтр)

Это отсеивает заведомо битые сборки без запуска тяжёлого SW.

**3.1. Сборка читается как .NET assembly:**
```powershell
powershell -NoProfile -Command "[Reflection.Assembly]::LoadFile('%RUNDIR%\ZTool.dll').FullName"
```
Ожидаемо: печатает `ZTool, Version=1.0.0.0, ...`. Если бросает исключение — сборка битая (FAIL на этом шаге).

**3.2. Регистрация COM/аддона (от администратора):**
```bat
cd /d %RUNDIR%
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\RegAsm.exe ZTool.dll /codebase
```
Ожидаемо: `Types registered successfully`. Любая ошибка — записать дословно.

**3.3. Ключи реестра аддона** (если RegAsm не создал/нужно вернуть «русский вид»):
```
HKEY_LOCAL_MACHINE\SOFTWARE\SolidWorks\Addins\{59959DFA-3229-4B86-852E-52ABF2BDB8C0}
    (Default)    REG_DWORD = 1
    Description  REG_SZ    = ZTool - инструменты для SolidWorks
    Title        REG_SZ    = ZTool
HKEY_CURRENT_USER\Software\SolidWorks\AddInsStartup\{59959DFA-3229-4B86-852E-52ABF2BDB8C0}
    (Default)    REG_DWORD = 1
```

## 4. Запуск в SolidWorks и проверка вкладки

1. Запусти SolidWorks 2025.
2. Дождись загрузки. Если выскочило окно ошибки аддона/.NET при старте — **сними скриншот и
   запиши текст дословно** (это главный признак «DLL не загрузилась»).
3. Проверь, что аддон включён: `Инструменты → Надстройки` (Tools → Add-Ins) — строка `ZTool`
   должна быть с галочкой. Отметь: есть ли строка, ставится ли галочка, какой текст (рус/кит).
4. Открой тестовую сборку из ASCII-пути, напр.
   `D:\...\TestModel\0614-A00.SLDASM`.
5. Найди вкладку **ZTool** на ленте (CommandManager). Открой её.

Для **каждого** элемента вкладки/панели зафиксируй язык (Русский / Китайский / Пусто-крякозябры):

| Элемент | Ожидаемый RU | Факт (RU/CN/битый) |
|---|---|---|
| Кнопка 1 | Переименование деталей | |
| Кнопка 2 | Сохранить выбранное | |
| Кнопка 3 | Случайная окраска | |
| Кнопка 4 | Управление файлами | |
| Кнопка 5 | Пакетная печать | |
| Кнопка «О программе» | О программе | |
| Всплывающие подсказки (tooltip) | — | |
| Заголовок панели свойств (PMP) | — | |

## 5. Проверка функциональности (паритет с оригиналом)

Локализация не должна менять поведение. Прогони базовые действия и отметь, работают ли они как
в оригинале:

| Функция | Действие | Результат (OK / ошибка) |
|---|---|---|
| Открытие панели | клик по кнопке ZTool | |
| Переименование деталей | запустить на `0614-A00` | |
| Форма SN/QR (`Frm_SN`) | открыть | |
| Экспорт/BOM (если есть в DLL) | запустить | |
| Закрытие/повторное открытие SW | перезапуск | |

Если любая функция падает с ошибкой — дословный текст ошибки + скриншот.

---

## 6. Откат (вернуть оригинал / снять кандидата)

```bat
cd /d %RUNDIR%
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\RegAsm.exe ZTool.dll /unregister
```
При необходимости верни оригинальный `ZTool.dll` (SHA256 из раздела 0) и перерегистрируй его.
SolidWorks при следующем старте подхватит исходный аддон.

## 8. Как мы итерируемся

1. Я присылаю кандидат `ZTool.dll` (+ при необходимости спутники) и его SHA256.
2. Ты кладёшь его в `%RUNDIR%`, проходишь разделы 3→5, заполняешь раздел 7, шлёшь отчёт + скриншоты.
3. По отчёту я правлю распаковку/русификацию и присылаю следующую сборку.
4. Цель — вердикт «полностью русская, функции работают». До него — повторяем цикл.

Самые ценные для меня данные при провале: **точный текст любой ошибки** (шаги 3.2, 4.2, 5) и
**скриншот вкладки** (видно, что русифицировалось, а что нет).
