# From-source: Фаза 2 — лицензионный слой в исходниках + рантайм-проверка

Дата: 2026-06-20
Ветка: `devin/1781976868-phase2-from-source-licensing` (поверх PR #47 `client-src`)
Трек 2 (полная реконструкция), milestone из `SWTools_FROM_SOURCE_PLAN_RU.md`.

## Контекст

Фаза 1 (PR #47) завела канонический редактируемый исходник клиента в `client-src/`,
собирающийся в `ZTool.exe` с 0 ошибок. Лицензионная логика в этом дереве была
«как у вендора» (сырая декомпиляция), тогда как боевую сборку даёт реинжект-пайплайн
`client-core`, вшивающий в exe **owned**-версии 4 классов лицензирования. Фаза 2 —
свести это в один источник правды (исходники) и проверить exe в рантайме.

## Что сделано

### 1. Перенос owned-версий лицензионных классов в `client-src`

`client-core/src` содержит 4 редактируемых класса, которые реинжектор транслирует в
exe. Они перенесены в `client-src/ZTool/` поверх вендорской декомпиляции:

| класс | изменение относительно сырой декомпиляции | поведение |
|-------|--------------------------------------------|-----------|
| `GetRegistrycreatedtime` | `RegCloseKey` для перечисляемого хэндла (вендорская декомпиляция течёт) | сохранено |
| `SR` | `using`/null-safe чтение реестра и освобождение хэндлов; поиск `ZToolARM.dll` дополнительно падает в родительский каталог | сохранено |
| `TCPClient` | лимит размера тела ответа (`MAX_RESPONSE_BODY`) + таймаут чтения; китайский заголовок `MessageBox` `提示` → «Сообщение» | сохранено + локализация |
| `SecurityCenter` | без изменений (идентичен донору) | — |

Все правки — hardening, сохраняющий поведение. Файлы 1:1 совпадают с донорами
`client-core/src`, так что исходник и реинжект больше не расходятся в лицензионной части.

### 2. Сборка из исходников

```
cd client-src && dotnet build ZTool.csproj -c Release --no-incremental
=> Build succeeded. 0 Error(s), 123 Warning(s)
```

123 warning — те же безобидные артефакты декомпиляции в остальном дереве (недостижимый
код после `goto`/`return`, неиспользуемые поля дизайнера).

## Рантайм-проверка (standalone, без SolidWorks)

Рантайм-папка: вывод сборки + рантайм-DLL из репо; `SWToolsARM.dll`→`ZToolARM.dll`,
`SWTools.dll`→`ZTool.dll`, `SWTools.settings`→`ZTool.settings`.

| проверка | результат |
|----------|-----------|
| `ZTool.exe` стартует, проходит анти-тампер гейт `SR.Isme` (MD5 `ZToolARM.dll` == `Resources.G_D5V`) | PASS |
| Рендерится полный клиент: лента «Главная/Спецификация/Упаковка/Инструменты» на русском | PASS |
| `FrmAbout` («Действующая лицензия не обнаружена», кнопки «Проба»/«Регистр»/«Отмена») открывается | PASS |
| `FrmRverify` (онлайн-активация: рег.код, «Активация онл», «Перенести») открывается | PASS |
| Аппаратная привязка в `FrmRverify` сообщает «Это устройство не соответствует условиям регистрации!» | PASS (штатное поведение на облачной VM с обнулённым UUID) |

Вывод: исходник функционально воспроизводит вендорский клиент, лицензионный UI
(триал/регистрация/онлайн-активация) доступен и работает из исходников.

## Найдено для Фазы 3 (рекоменд. следующий шаг)

Из исходников всё ещё проявляется вендорское поведение, которое в боевом exe убирает
`Localizer`/реинжект:

1. **Сетевой апдейт-чек** на старте лезет на `z-tool.cn` и показывает окно «проверка
   обновлений» с **китайским** текстом (`发布日期`, `1.[新增]…`). В боевой сборке авто-апдейт
   отключён — нужно перенести отключение/локализацию в исходники.
2. `FrmAbout`: 4 QR-кода с китайскими подписями (`淘宝/公众号/QQ群/抖音`), вендорские
   контакты (`z-tool.cn`, QQ, Taobao), логотип-кубик — требуется ребрендинг SWTools.
3. `FrmRverify`: заголовок окна `注册` и плейсхолдер пароля `请输入8-20位…` — остаточный
   китайский, переводится `Localizer`-ом; перенести в `.resx`/исходники.

## Замечания

- SolidWorks на стенде нет — проверки, требующие SW (главное окно с моделью, экспорт BOM,
  add-in), остаются за агентом на SW-машине; здесь они вне области Фазы 2.
- Полный серверный хендшейк активации (`Активация онл` → сервер лицензий) требует
  перенаправленного на наш сервер эндпоинта (rekey) и валидного железа; на облачной VM
  блокируется штатной аппаратной проверкой до сетевого обмена.
