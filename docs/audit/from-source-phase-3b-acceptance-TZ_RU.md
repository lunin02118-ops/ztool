# ТЗ на приёмочную проверку — Фаза 3b (ребренд SWTools + русификация из исходников)

Для: агент на машине с **SolidWorks 2025**.
Объект: `ZTool.exe`, собранный из `client-src` (ветка
`devin/1782000000-phase3-rebrand-localization`), а также — для сверки 1:1 —
боевой `SWTools.exe` из бинарного пайплайна.

Цель: подтвердить, что сборка из исходников ведёт себя как боевой реинжект:
ребренд SWTools, нет китайского, нет вендорского апдейт-чека, корректные окна,
рабочая лицензия и BOM-экспорт на реальных данных SolidWorks.

## Подготовка стенда

1. Собрать клиент: `dotnet build client-src/ZTool.csproj -c Release`
   → `client-src/bin/Release/net48/ZTool.exe`.
2. Рядом с exe положить нативную `SWToolsARM.dll` (как ставит инсталлятор).
3. Установить SolidWorks 2025, открыть тестовую сборку с BOM.
4. Очистить реестр от прошлых прогонов: проверить, что нет
   `HKCU\SOFTWARE\ZTool` и `HKCU\SOFTWARE\SWTools` (или зафиксировать исходное
   состояние).

> **Решение по совместимости (от заказчика): только чистая установка.**
> Полный уход от китайского — осознанный: старые конфиги с китайскими
> операторами фильтров (`等于`, `包含`…) и типами свойств НЕ мигрируются.
> Путь обновления поверх старого ZTool не поддерживается; стенд проверяет
> сценарий чистой установки. Авто-конвертация старых конфигов не требуется.

## Сценарии

### S1. Ребренд и отсутствие китайского (standalone)
- Запустить exe. Ожидается: заголовок «SWTools 1.0(x64)», иконка — лого с сайта.
- Пройти по ленте (Главная/Спецификация/Упаковка/Инструменты) и меню File.
- **PASS:** нигде нет иероглифов; весь видимый текст русский; иконка SWTools.

### S2. Окно «О программе»
- File → «О программе».
- **PASS:** заголовок «О программе SWTools — …»; лого-баннер SWTools; ссылки
  «Перейти на сайт» и «Связаться в MAX» открывают `license.vizbuka.ru/swtools/`
  и MAX соответственно; строка Email; MAX-QR; кнопка OK; вендорских контактов
  (z-tool.cn, QQ, Taobao) и китайского нет.

### S3. Окно «Лицензия не обнаружена» (триал)
- Запустить без лицензии (или «Демо»).
- **PASS:** компактный баннер «Лицензия не обнаружена»; Email; «Перейти на сайт»;
  MAX-QR; кнопки «Демо»/«Регистрация»/«Отмена»; вендорских грид-контактов нет.

### S4. Апдейт-чек отключён
- При старте — убедиться, что нет вендорского окна проверки обновлений.
- File → «Проверить обновления».
- **PASS:** не открывается окно/браузер, нет обращения к z-tool.cn, нет
  китайского текста (поведение инертно).

### S5. Активация / перенос лицензии против БОЕВОГО сервера лицензий  ← реальное железо + донгл

> **Почему только на реальном железе.** Клиент сначала проходит аппаратную
> анти-тампер-проверку (`SR.Isme` / `SR.GetMNum`, отпечаток железа/MNum) и лишь
> затем открывает TCP-сокет к серверу. На облачной/виртуальной машине проверка
> не проходит — exe завершается (exit 0) ещё ДО сетевого обмена (подтверждено:
> так же ведёт себя и боевой `SWTools.exe`). Поэтому серверные сценарии S5/L*
> выполняются **только на реальной лицензируемой машине с подключённым донглом**.

**Протокол (для понимания, что проверяем).** Адрес и порт боевого сервера зашиты
RSA-шифром в `TCPClient.Connect()` (`TCPClient.cs:110-111`) и расшифровываются в
рантайме — клиент сам идёт на боевой сервер, отдельной настройки адреса нет.
Обмен — двухстадийный Apply→Confirm. Типы запросов (`TCPClient.Sendtype`):
`Apply_register=128` (заявка на активацию), `Apply_Remove` (заявка на снятие/перенос),
`register`, `verify_register` (проверка действующей лицензии при старте,
`sendstring(verify_register, sR.get_rginfo())`). Активированный ключ пишется в
`HKCU\SOFTWARE\SWTools`, значение `sn` (`FrmRg.cs:1025-1026`).

**Подготовка серверных сценариев.**
1. Реальная машина с валидным железом + подключённый донгл.
2. Доступна сеть до боевого сервера лицензий (если за NAT/файрволом — открыть
   исходящий TCP на порт сервера; при возможности снять `pcap` на время обмена).
3. На стороне сервера — включить логирование обмена (см.
   `docs/.../phase-01-secrets-logging`), чтобы сверить принятые кадры.
4. До прогона: экспортировать `HKCU\SOFTWARE\ZTool` и `HKCU\SOFTWARE\SWTools`
   (`reg export`), зафиксировать исходное состояние БД лицензий на сервере.

**Процедура «чистого стенда» для L2 (обязательна перед активацией).**
Сценарий — clean install, поэтому остатки прошлых прогонов в реестре дают ложный
результат (клиент видит старый `sn` и не идёт на сервер). Перед L2:
```cmd
:: 1) бэкап (для восстановления после теста)
reg export "HKCU\SOFTWARE\SWTools" %TEMP%\bak_swtools.reg /y
reg export "HKCU\SOFTWARE\ZTool"   %TEMP%\bak_ztool.reg   /y
:: 2) удалить ключи лицензии обоих брендов
reg delete "HKCU\SOFTWARE\SWTools\sn" /f
reg delete "HKCU\SOFTWARE\ZTool\sn"   /f
:: 3) убедиться, что ключей нет (оба query должны вернуть "не удалось найти")
reg query "HKCU\SOFTWARE\SWTools\sn"
reg query "HKCU\SOFTWARE\ZTool\sn"
```
После теста — при необходимости вернуть исходное состояние из `*.reg`-бэкапов.
**Только теперь** L1 на этой машине должен показывать «Лицензия не обнаружена»
(а не молчаливый старт по старому `sn`), и L2 будет настоящей онлайн-активацией.

#### L1. Стартовая проверка лицензии (`verify_register`)
- Запустить from-source exe на машине с уже действующей лицензией.
- **PASS:** клиент соединяется с боевым сервером, отправляет `verify_register`,
  сервер подтверждает — главное окно открывается без окна регистрации; в
  серверном логе видна запись `verify_register` от этого MNum.

#### L2. Онлайн-активация (`Apply_register`, кнопка «Активация онлайн»)
- Без лицензии: открыть «Регистрация» → ввести регистрационный код → «Активация онлайн».
- **PASS:** проходит двухстадийный Apply→Confirm; в `HKCU\SOFTWARE\SWTools\sn`
  появляется ключ (НЕ под `ZTool`); на сервере заводится/занимается слот для этого
  MNum; после активации клиент стартует с действующей лицензией (повтор L1 = PASS).

#### L3. Перенос / снятие лицензии (`Apply_Remove`, кнопка «Перенести лицензию»)
- На активированной машине: «Регистрация» → «Перенести лицензию».
- **PASS:** проходит Apply→Confirm на снятие; ключ из `HKCU\SOFTWARE\SWTools`
  очищается; слот на сервере освобождается; лицензию можно активировать заново
  (повтор L2 на этой же или другой машине = PASS).

#### L4. Негативные сценарии
- Неверный регистрационный код; обрыв сети в середине Apply→Confirm; повторная
  активация уже занятого MNum.
- **PASS:** клиент показывает понятную русскую ошибку, не падает, не пишет
  «полуактивированный» ключ; состояние сервера консистентно (слот не теряется).

#### L5. Сверка протокола from-source ⇄ боевой `SWTools.exe`
- Снять кадры обмена (pcap/серверный лог) для L1–L3 сначала на боевом `SWTools.exe`,
  затем на from-source exe (на той же машине/донгле).
- **PASS:** последовательность типов кадров, размеры и реакции сервера идентичны;
  расшифрованный адрес/порт сервера совпадает; различий в хендшейке нет.

### S6. Ярлык рабочего стола
- В Параметрах создать ярлык на рабочем столе.
- **PASS:** создаётся `SWTools.lnk` (НЕ `ZTool.lnk`), указывает на exe.

### S7. Живой BOM-экспорт (главный сценарий) ← требует SW-машину
- Открыть сборку, сформировать спецификацию, экспортировать в Excel-шаблон.
- **PASS:** экспорт проходит как в боевой сборке; Author документа = «SWTools»;
  колонки заполняются по русским defined names (см. ТЗ Фазы 11); китайского в
  результате нет.

### S8. Сверка 1:1 с боевым `SWTools.exe`
- Прогнать S1–S7 на боевом `SWTools.exe` (бинарный реинжект) и сравнить с
  from-source сборкой.
- **PASS:** видимое поведение и UI идентичны.
- Примечание: from-source имеет одно намеренное улучшение читаемости —
  пробелы в статус-строках («Всего 5 файлов» вместо «Всего5файлов»). Это
  единственное допустимое расхождение с боевым exe (в боевом — слитно из-за
  несогласованного `translations.tsv`); НЕ считать за FAIL.

### S9. Имена каталогов и переведённые пути ← требует развёрнутый инсталлятор
- Проверить, что инсталлятор создаёт каталоги ровно с теми именами, которые
  использует клиент: «Шаблоны спецификации» (BOM-шаблоны), «Шаблон SW»
  (база материалов, `FrmOptions`), «Изображение» (`FrmOutputoptions`).
- Открыть диалоги «Обзор…» в экспорте BOM, в настройках материалов и вывода.
- **PASS:** каждый диалог открывается в правильном существующем каталоге, а не
  в каталоге-фолбэке.

### S10. Кнопка «Параметры печати» на русской Windows ← требует SW-машину
- В окне печати нажать кнопку настроек печати принтера («Параметры печати»).
- **PASS:** открывается системный диалог свойств принтера. **FAIL/заметка:**
  если кнопка не реагирует — verb сопоставляется по локализованному имени
  (`"Параметры печати (&E)..."`), которое может отличаться по версии Windows;
  зафиксировать фактическое имя verb на стенде для последующей доработки.

## Отчёт
По каждому сценарию: PASS/FAIL, скриншот, заметки. FAIL по S5/S7/S8 эскалировать
с логами (реестр, ответ сервера лицензий, файл-результат BOM).
