# ТЗ приёмки на SolidWorks 2025 — BOM-экспорт (Phase 11)

Документ для агента/инженера на машине с установленным **SolidWorks 2025** и
рабочей лицензией ZTool/SWTools. Цель — подтвердить, что после Phase 11
BOM-экспорт работает 1:1, китайского нигде нет, и три UX/стабильностных дефекта
закрыты. Это ручная приёмка: соберите `SWTools.exe` из этой ветки, подложите в
рабочую папку рантайма и пройдите сценарии ниже, фиксируя результат и скриншоты.

---

## 0. Подготовка стенда

1. Соберите клиент из ветки PR (на Windows с .NET SDK 10 + 8):
   ```powershell
   $env:PATH = 'C:\dotnet;' + $env:PATH   # или ваш путь к dotnet
   cd client-core
   .\build.ps1
   ```
   Ожидаемо: `dangling typerefs = 0`, в `AssemblyRefs` **нет** `ZBinderDonor`,
   `BinderInject verify: PASS`. Артефакт: `client-core\out\SWTools.exe`.
2. Скопируйте `client-core\out\SWTools.exe` поверх `SWTools.exe` в **полной**
   папке рантайма (рядом с остальными DLL и `SWTools.settings`). Не запускайте
   из `out\` — там нет зависимостей.
3. Убедитесь, что используется `SWTools.settings` и
   `Шаблоны спецификации\bom_шаблон.xlsx` из этой ветки (русские `mappingname`
   и defined names). Путь к шаблону в settings (`bomname`) должен указывать на
   этот `bom_шаблон.xlsx`.
4. Подготовьте тестовую сборку SolidWorks (или используйте существующую), у
   деталей которой заполнены кастомные свойства с **русскими** именами,
   совпадающими с заголовками колонок (см. таблицу в §2). Минимум: несколько
   деталей с `Наименование`, `Обозначение`, `Материал`, `Тип`, `Версия`,
   `ОбработкаПоверхности`; хотя бы одна покупная (`Тип = Покупное`) и одна
   обрабатываемая (`Тип = Мех.обработка`).

---

## 1. Smoke: интерфейс без китайского

**Шаги.** Запустите SWTools в SolidWorks, откройте основное окно ZTool и
диалоги: «Экспорт спецификации», «Сопоставление», «Задать имя свойства»
(кнопка «Импорт…»), «Опции».

**Ожидаемо (PASS).**
- Нигде в UI нет китайских иероглифов.
- Заголовки колонок таблицы и пунктов меню — на русском.

---

## 2. BOM-экспорт: заполнение всех русских колонок

**Шаги.**
1. Постройте/обновите спецификацию по тестовой сборке.
2. Выполните экспорт в Excel-шаблон (`bom_шаблон.xlsx`).
3. Откройте полученный `.xlsx`, лист **«Спецификация»**.

**Проверка соответствия колонка → defined name → ячейка (строка данных = 6+).**

| Колонка ZTool | Defined name шаблона | Якорь | Источник значения |
|---|---|---|---|
| Номер п/п | `Номер` | A6 | автонумерация |
| Наименование | `Наименование` | C6 | свойство `Наименование` |
| Обозначение | `Обозначение` | D6 | свойство `Обозначение` |
| Версия | `Версия` | E6 | свойство `Версия` |
| Ед. изм. | `ЕдИзм` | F6 | служебное |
| Кол-во | `Количество` | G6 | расчётное |
| Тип | `Тип` | H6 | свойство `Тип` |
| Обработка поверхности | `ОбработкаПоверхности` | I6 | свойство |
| Масса ед., кг | `МассаЕдКг` | J6 | расчётное (`Col_Weight`) |
| Имя файла | `ИмяФайла` | L6 | служебное |
| Эскиз | `Эскиз` | M6 | служебное |
| Материал | `Материал` | N6 | свойство `Материал` |
| Путь | `Путь` | O6 | служебное |
| Габаритные размеры | `ГабаритныеРазмеры` | P6 | расчётное (`Col_bound`) |

**Ожидаемо (PASS).**
- Каждая заполненная колонка попала в правильный столбец (A…P) по defined name.
- Значения свойств деталей совпадают с тем, что в дереве/таблице ZTool.
- Расчётные `Масса ед., кг` и `Габаритные размеры` заполнены (не пустые там, где
  есть геометрия/материал).
- В выгрузке нет китайских строк и нет «съехавших» на соседний столбец данных.

**FAIL-индикаторы.** Данные в неверном столбце; пустые колонки при наличии
свойства; китайские заголовки/значения; ошибка при экспорте.

---

## 3. «Сопоставление»: все расчётные/служебные колонки

**Шаги.** Откройте диалог «Сопоставление» (ручной маппинг колонок BOM на
defined names шаблона).

**Ожидаемо (PASS).**
- В списке присутствуют **все** служебные/расчётные колонки `Col_*`
  (в т.ч. `Масса ед., кг` и `Габаритные размеры`, ранее единственные две), кроме
  нерелевантных служебных (чекбокс выбора строки, «Новая папка»).
- Каждую такую колонку можно вручную сопоставить с defined name шаблона, и после
  сопоставления + экспорта значение попадает в назначенную ячейку.

**Регрессионная проверка.** Сопоставьте, например, `Габаритные размеры` на
свободный defined name и подтвердите, что экспорт пишет туда.

**FAIL-индикаторы.** В списке только 2 расчётные колонки; ручной маппинг
расчётной колонки не влияет на экспорт.

---

## 4. Якорь кнопки «Импорт…» (диалог «Задать имя свойства»)

**Шаги.**
1. Откройте «Задать имя свойства» (Frmsetpropname) — там кнопка «Импорт…».
2. Запомните её положение (левый-нижний угол).
3. Растяните диалог за угол (он resizable): увеличьте ширину и высоту.

**Ожидаемо (PASS).** Кнопка «Импорт…» остаётся «приклеенной» к левому-нижнему
углу формы (anchor `Bottom|Left`) и не уплывает к центру/верху при ресайзе.

**FAIL-индикаторы.** Кнопка дрейфует к центру или остаётся у верхнего края при
увеличении формы.

---

## 5. Десериализация на «Импорт» (ошибка `System.Object из ZTool…`)

**Контекст.** Ранее «Импорт» (загрузка сохранённой конфигурации/набора имён
свойств, содержащих `DataTable`/`Font`/`Color`-блобы, записанные другой
версией рантайма) падал с `System.Object из ZTool…`
(`SerializationException`).

**Шаги.**
1. В «Задать имя свойства» → «Импорт…» загрузите ваш реальный
   `SWTools.settings`/набор из `D:\SWTools\` (тот, на котором воспроизводилась
   ошибка), либо конфиг, ранее вызывавший сбой.
2. Аналогично проверьте импорт/вставку из буфера в окнах со списками
   (Output/Print/Drw/SyncDrwName) — они используют тот же безопасный binder.

**Ожидаемо (PASS).** Импорт проходит без `System.Object из ZTool…` и без иных
`SerializationException`; данные подгружаются корректно.

**FAIL-индикаторы.** Любое исключение десериализации / сообщение
`System.Object из ZTool…` / `Could not load file or assembly 'ZBinderDonor'`.

> Если у вас есть конкретный падавший `SWTools.settings` — приложите его к отчёту
> о приёмке; это эталонный воспроизводящий артефакт для регресс-набора.

---

## 6. Фильтры по «Тип» (русские значения)

**Контекст.** Из правил `FilterRulesList` удалены китайские токены `机加`/`外购`;
остались русские значения.

**Шаги.** Примените пресеты/правила «Обрабатываемые детали», «Покупные изделия»,
«С обработкой поверхности» к тестовой сборке.

**Ожидаемо (PASS).**
- «Обрабатываемые детали» включает детали с `Тип ∈
  {Мех.обработка, Листовая, Литьё, Сварка}`.
- «Покупные изделия» включает детали с `Тип ∈ {Покупное, Стандартное}`.
- Фильтр не «пропускает всё подряд» (правило не игнорируется).

**FAIL-индикаторы.** Фильтр ничего не отбирает или отбирает все строки
(признак сломанного формата правила).

---

## Форма отчёта о приёмке

Для каждого раздела §1–§6 укажите `PASS`/`FAIL`, версию SolidWorks/ZTool,
SHA256 проверенного `SWTools.exe`, и приложите скриншоты:
- UI без китайского (§1);
- заполненный экспортированный `.xlsx` (§2);
- список колонок в «Сопоставлении» (§3);
- диалог «Импорт…» до/после ресайза (§4);
- успешный «Импорт» падавшего ранее конфига (§5).

Любой `FAIL` — со скриншотом и, по возможности, входным артефактом (сборка/конфиг),
чтобы воспроизвести оффлайн.
