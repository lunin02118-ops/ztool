# Восстановление исходников ZTool/SWTools — мастер‑карта (отправная точка)

> Это **единый отправной документ** по задаче «восстановление исходников»: что уже
> сделано, где мы сейчас и что осталось. Детальный поэтапный план путей сборки
> (A→E) живёт в [`from-source-roadmap_RU.md`](from-source-roadmap_RU.md); здесь —
> верхнеуровневая картина со статусами и ссылками на PR.

## 1. Цель

Сделать каталог исходников `client-src` **единственным источником правды**, из
которого можно легко править, модифицировать и быстро собирать/проверять новые
релизы, и постепенно вывести из эксплуатации хрупкий бинарный IL‑реинжект
(`client-core/tools/Reinjector` + `Localizer` + `BinderInject`), которым раньше
«лечили» готовый чужой бинарь.

Критерий завершения задачи:
- весь клиент (EXE **и** SolidWorks‑аддин) собирается из `client-src` одной командой;
- CI собирает from‑source и гоняет приёмочные гейты на собранном артефакте;
- релиз/инсталлятор берут артефакты из from‑source сборки;
- в UI нет китайского; внутренний китайский сведён к неизменяемым крипто‑константам;
- боевой прогон (лицензии L1–L5 + BOM) на реальном железе совпадает 1:1 с поведением
  принятого боевого `SWTools.exe`.

## 2. Два пути сборки (как есть сейчас)

| Путь | Что делает | Состояние |
|------|------------|-----------|
| **IL‑реинжект** (`client-core/build.ps1` + `Reinjector`/`Localizer`/`BinderInject`) | Патчит/локализует/хардирует готовый чужой бинарь | Исторический — выводится на Этапе E |
| **from‑source** (`client-src/ZTool.csproj`, `WinExe`, net48) | Компилирует клиент из исходников | Основной; пока **только EXE**, аддина в исходниках ещё нет |

## 4. Текущее состояние `main` (факт)

- `client-src` собирается локально и в CI (`dotnet build client-src/ZTool.csproj -c Release` → 0 ошибок), бренд **SWTools**, версия `1.1.4`.
- В `client-src` есть **EXE** и своё лицензионное ядро в исходниках. SolidWorks‑аддин
  (`ZTool.dll`/`SwAddin`) и `Ribbon.dll` пока **пребилт‑артефакты** (исходников аддина нет).
- Приёмочные гейты и инварианты строк гоняются на каждый PR по from‑source.
- Остаётся «несущий», невидимый в UI китайский: имена шрифтов (`微雅` и др.), 6 крипто‑пассфраз,
  семантический ключ `零` (тег `Col_Extname`).
- Боевая приёмка на реальном железе: **S2 закрыт и подтверждён**; **S7 (BOM)** и **L1–L5 (лицензии)**
  ещё не пройдены вживую (нужен COM‑зарегистрированный аддин и реальный донгл).

## 5. Что осталось

### 5.1 Срочное — валидация открытых правок
- **#57 (десериализация):** подтвердить на боевой машине с SolidWorks, что «Импорт… →
  Получить из файла», «Столбец заполнения» и «Параметры вывода» больше не падают. Корень
  — `DataTable`‑колонка типа `System.Object`, сериализованная с идентичностью
  `…f08fc7047657204e`; на собранном exe фикс подтверждён детерминированно. После подтверждения — merge.

### 5.2 Этап D — фаза аддина (декомпозиция `ZTool.dll`/`SwAddin`/`Ribbon.dll` в исходники)
- Перенести SolidWorks‑аддин в `client-src` (отдельный csproj), как сделали с exe; реф — PR #43.
- Закрывает **последний бинарный артефакт** → весь клиент собирается из исходников.
- Здесь же планово переименовать семантический ключ `零` (`Col_Extname`) **разом с консьюмером
  в аддине** (половинчатый перевод и был источником бага «случайный цвет»).
- Закрывает корень **S7** (незагруженный/не COM‑зарегистрированный аддин): with исходниками
  можно собрать, подписать и зарегистрировать аддин управляемо.
- *На паузе до результатов боевого прогона L1–L5* (решение пользователя).

### 5.3 Этап E — переключить релиз/инсталлятор на from‑source и вывести IL‑реинжект
- Пакет/инсталлятор берут exe **и аддин** из from‑source сборки.
- IL‑реинжект (`Reinjector`/`Localizer`/`BinderInject`) → в архив после паритета 1:1.

### 5.4 Чистка остаточного китайского (по мере владения исходниками)
- Имена шрифтов `微雅`/`华行`/`宋`/`楷` → латинское семейство (помечены `REVIEW`, не блокируют).
- Китайские *идентификаторы* (имена enum/контролов) — косметика; в тексте не всплывают.
- 6 крипто‑пассфраз — **остаются навсегда** (их перевод/удаление ломает хендшейк лицензий),
  зафиксированы в `source_required_keys.tsv`.

### 5.5 Боевой прогон и отчётность (на реальном железе)
- Прогон строго по `docs/release/BOEVOY_E2E_TEST_RUNBOOK_RU.md` и наряду
  `docs/release/EXECUTOR_TEST_TASKING_RU.md`: голая установка → полная очистка реестра обоих
  брендов → триал → **L1–L5** → BOM → сверка 1:1 с боевым exe.
- Перед SW обязательно: skill `solidworks-ztool-testing` + проверка загрузки аддина (S7‑gate).
- Результат: отчёт `docs/audit/from-source-boevoy-acceptance-report_<дата>_RU.md` (таблица
  S1–S10/L1–L5 = PASS/FAIL + доказательства).

## 6. Открытые вопросы / риски
- **Аддин не в исходниках** — главный технический долг; блокирует S7, BOM и полный паритет.
- **L1–L5 не пройдены вживую** — лицензионный поток подтверждён только локально/частично.
- **Остаточный китайский** — снимается по мере замены строк; гейт инвариантов не даст вернуться.
- **Два пути сборки сосуществуют** — до Этапа E легко перепутать источник; релиз пока завязан на IL‑путь.
