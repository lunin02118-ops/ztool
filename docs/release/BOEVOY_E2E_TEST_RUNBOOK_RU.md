# Сквозной runbook боевого теста SWTools (чистая установка → спецификация/BOM)

Для: агент-исполнитель на **реальной машине с SolidWorks 2025** (валидное железо + донгл).
Объект: `ZTool.exe` (бренд **SWTools**), собранный из `client-src`, рядом — нативная
`SWToolsARM.dll`. Для сверки 1:1 — боевой `SWTools.exe` из бинарного пайплайна.

Это **операторский сценарий по шагам** в правильном порядке: от голой установки и очистки
реестра до генерации ведомости материалов (спецификации/BOM). Подробные критерии PASS/FAIL
по каждому окну — в `docs/audit/from-source-phase-3b-acceptance-TZ_RU.md` (сценарии S1–S10,
L1–L5); здесь — сквозная последовательность и сбор доказательств.

> **Решение по совместимости (заказчик): только чистая установка.** Миграция старых
> конфигов с китайскими операторами фильтров (`等于`, `包含`…) не поддерживается.

---

## Этап 0. Подготовка и фиксация исходного состояния
1. Зафиксировать сборку: `dotnet build client-src/ZTool.csproj -c Release`
   → `client-src/bin/Release/net48/ZTool.exe`. Записать `ProductVersion` exe.
2. Подготовить регистрационный код (реальный) и убедиться, что донгл подключён.
3. Сеть до боевого сервера лицензий доступна (адрес зашит в клиенте; отдельной настройки нет).
   Если за NAT/файрволом — открыть исходящий TCP на порт сервера; по возможности подготовить
   захват `pcap` на время обмена.
4. На стороне сервера — включить логирование обмена и зафиксировать исходное состояние БД
   лицензий (для сверки слота до/после).

## Этап 1. Бэкап и полная очистка реестра (чистый стенд)
Цель — гарантировать, что клиент не стартует по остаткам прошлых прогонов.
```cmd
:: 1) бэкап всех веток обоих брендов (для восстановления после теста)
reg export "HKCU\SOFTWARE\SWTools" %TEMP%\bak_swtools.reg /y 2>nul
reg export "HKCU\SOFTWARE\ZTool"   %TEMP%\bak_ztool.reg   /y 2>nul

:: 2) удалить ВСЕ ветки обоих брендов целиком (не только sn)
reg delete "HKCU\SOFTWARE\SWTools" /f 2>nul
reg delete "HKCU\SOFTWARE\ZTool"   /f 2>nul

:: 3) проверить, что веток нет (оба query должны вернуть "не удалось найти")
reg query "HKCU\SOFTWARE\SWTools"
reg query "HKCU\SOFTWARE\ZTool"
```
**Доказательство:** скриншоты обоих `reg query` с «не удалось найти».
После теста при необходимости восстановить из `*.reg`-бэкапов.

## Этап 2. Установка «с нуля» (голая программа)
1. Развернуть инсталлятор/пакет на чистой машине (или положить exe + `SWToolsARM.dll` +
   каталоги-ресурсы, как ставит инсталлятор).
2. Проверить, что созданы каталоги ровно с теми именами, что использует клиент:
   «Шаблоны спецификации» (BOM-шаблоны), «Шаблон SW» (база материалов), «Изображение»
   (соответствует сценарию **S9**).
3. Зарегистрировать аддин в SolidWorks (как делает инсталлятор), запустить SolidWorks 2025.
   - До декомпозиции аддина (Этап D дорожной карты) аддин остаётся бинарным (`SWTools.dll`):
     его нужно COM-зарегистрировать (`regasm SWTools.dll /codebase`) и завести ключи
     `SOLIDWORKS\AddIns\{GUID}` — простого копирования DLL рядом с exe **недостаточно**.
   - **Обязательная проверка загрузки (gate перед Этапом 9):** убедиться, что аддин реально
     поднялся в SolidWorks — `GetAddInObject("SWTools.SwAddin")` возвращает объект (не
     `Element not found / 0x8002802B`), а вызов «Подключить SW» наполняет таблицу деталей
     (`PartHits>0`). Если объект аддина не найден или таблица пустая — **стоп**: BOM (S7)
     невалиден, эскалировать (это сорвало прогон 2026-06-21).

## Этап 3. Первый запуск без лицензии (триал) — `S1`, `S3`, `S4`
1. Запустить SWTools. Заголовок «SWTools 1.0(x64)», иконка SWTools, весь UI русский,
   нет иероглифов (**S1**).
2. Окно «Лицензия не обнаружена»: компактный баннер, Email, «Перейти на сайт», MAX-QR,
   кнопки «Демо»/«Регистрация»/«Отмена»; вендорских контактов нет (**S3**).
3. Апдейт-чек инертен: ни при старте, ни по «Проверить обновления» нет окна/браузера/обращения
   к z-tool.cn (**S4**).
**Доказательство:** скриншоты главного окна, «Лицензия не обнаружена», «О программе» (**S2**).

## Этап 4. Стартовая проверка лицензии — `L1`
- На machine без активной лицензии клиент при старте должен показать «Лицензия не обнаружена»
  (а не молчаливый старт по старому `sn` — это и проверяет чистый стенд из Этапа 1).
- На уже активированной машине: при старте клиент шлёт `verify_register` на боевой сервер,
  сервер подтверждает, главное окно открывается без окна регистрации; в серверном логе —
  запись `verify_register` от этого MNum.

## Этап 5. Онлайн-активация — `L2`
1. «Регистрация» → ввести реальный регистрационный код → «Активация онлайн».
2. **PASS:** проходит двухстадийный Apply→Confirm; в `HKCU\SOFTWARE\SWTools\sn` появляется
   ключ (**НЕ** под `ZTool`); на сервере заводится/занимается слот для MNum; после активации
   клиент стартует с действующей лицензией (повтор Этапа 4/L1 = PASS).
**Доказательство:** `reg query "HKCU\SOFTWARE\SWTools\sn"` (ключ есть), серверный лог/pcap
обмена, скриншот успешной активации.

## Этап 6. Перенос / снятие лицензии — `L3`
- На активированной машине: «Регистрация» → «Перенести лицензию».
- **PASS:** проходит Apply→Confirm на снятие; ключ в `HKCU\SOFTWARE\SWTools` очищается; слот
  на сервере освобождается; лицензию можно активировать заново (повтор L2 = PASS).

## Этап 7. Негативные сценарии — `L4`
- Неверный код; обрыв сети в середине Apply→Confirm; повторная активация занятого MNum.
- **PASS:** понятная русская ошибка, клиент не падает, не пишет «полуактивированный» ключ;
  состояние сервера консистентно (слот не теряется).

## Этап 8. Сверка протокола from-source ⇄ боевой exe — `L5`
- Снять кадры (pcap/лог) для L1–L3 сначала на боевом `SWTools.exe`, затем на from-source exe
  (та же машина/донгл). **PASS:** последовательность типов кадров, размеры, реакции сервера
  идентичны; расшифрованный адрес/порт совпадает.

## Этап 9. Генерация ведомости материалов (спецификация / BOM) — `S7`
0. **Предусловие:** аддин загружен (gate из Этапа 2.3): `GetAddInObject("SWTools.SwAddin")`
   даёт объект, «Подключить SW» вернул `PartHits>0`. Без этого экспорт BOM невалиден.
1. Открыть тестовую сборку в SolidWorks 2025.
2. Сформировать спецификацию (BOM) средствами SWTools, экспортировать в Excel-шаблон
   «Шаблоны спецификации\bom_шаблон.xlsx».
3. **PASS:**
   - экспорт проходит как в боевой сборке; в результате **нет китайского**;
   - Author/свойства документа = «SWTools»;
   - колонки заполняются по **русским** defined names (см. ТЗ Фазы 11,
     `docs/audit/phase-11-bom-export-acceptance-TZ_RU.md`): расчётные/служебные `Col_*`
     (МассаЕдКг, ГабаритныеРазмеры и т.д.) попадают в правильные ячейки;
   - окно «Сопоставление» показывает все `Col_*` без пустых строк;
   - кнопка «Импорт…» — у левого нижнего угла, «ОК»/«Отмена» — у правого; при сжатии не
     обрезаются.
4. (Опц.) Детерминированная сверка результата ассертером `bom_export_assert.py` против
   golden, если golden снят для этой сборки (включая RGB-цвет заливки ячеек).
**Доказательство:** экспортированный `.xlsx`, скриншоты «Сопоставления» и результата.

## Этап 10. Сверка 1:1 с боевым `SWTools.exe` — `S8`
- Прогнать Этапы 3–9 на боевом `SWTools.exe` и сравнить с from-source.
- **PASS:** видимое поведение и UI идентичны. Единственное допустимое расхождение —
  пробелы в статус-строках («Всего 5 файлов» vs «Всего5файлов») — НЕ FAIL.

---

## Отчёт
По каждому этапу: PASS/FAIL, скриншот/лог, заметки. Для лицензионных этапов (4–8) приложить
серверный лог/pcap и до/после-состояние `reg query` и слота на сервере. Для BOM (Этап 9) —
сам экспортированный файл. FAIL по лицензии/BOM/сверке эскалировать с полным набором логов.
