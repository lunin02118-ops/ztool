# FrmFilling layout visual gate

Дата: 2026-06-19.

Цель: проверить исправление формы `ZTool.FrmFilling` (`Столбец заполнения`) после жалоб на обрезанный текст, разные правила растягивания вкладок и невозможность нормально менять размер окна.

## Проверенный билд

- Источник: `client-core/dist/ZTool_binderfix.exe`
- Localizer: `client-core/tools/Localizer/bin/Release/net10.0/localize.dll`
- Версия: `1.1.0-alfa`
- Тестовый runtime: `_local_artifacts/frmfilling-layout-test/runtime`

## Метод

- Форма открывалась напрямую через WinForms reflection из тестового runtime.
- Вкладки выбирались через UI Automation `TabItem`, без координатных кликов.
- Дополнительно каждая вкладка открывалась отдельным запуском формы в размере `980x700`, чтобы проверить поведение при увеличенном окне.

## Результат

- Окно `FrmFilling` стало `Sizable`, минимум `760x540`.
- Вкладки `Пользовательское заполнение`, `Заполнение по правилу`, `Заполнение поиском`, `Связанное заполнение` читаемы в default и large вариантах.
- Исправлены слипшиеся подписи `Начальная строка`, `строка`, `записать в`.
- Таблица связанного заполнения оставлена на `Dock=Fill`; конфликт `Dock/Anchor` убран.

Визуальный лист проверки: `frmfilling-layout-contact-sheet.jpg`.

## Границы проверки

Это визуальный gate формы `FrmFilling`. Полный живой прогон SolidWorks/BOM/export этим отчётом не закрывается.
