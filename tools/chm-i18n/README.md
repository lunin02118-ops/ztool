# chm-i18n — перевод справки `help.CHM` на русский

Инструмент пересобирает русскую справку **`help_ru.chm`** (в корне репозитория)
из оригинальной китайской **`help.CHM`**. Перевод полностью воспроизводим:
словарь сегментов хранится в репозитории, бинарный CHM — лишь артефакт сборки.

## Состав

| Файл          | Назначение                                                            |
|---------------|-----------------------------------------------------------------------|
| `runs.json`   | 256 исходных китайских текстовых сегментов (порядок «от длинного к короткому») |
| `ru.json`     | их русские переводы, строго один-к-одному по индексу с `runs.json`    |
| `build_ru.py` | сборщик: перевод HTML-тем, навигации (HHC/HHK), генерация `.hhp`       |
| `check_chm_brand.py` | gate: проверяет собранный CHM на запрещённый legacy brand `ZTool` |
| `compare_manual_screenshots.py` | gate перед сборкой: покадровое сравнение оригинальных и RU-скриншотов |

## Как пересобрать

Требуется Python 3 и **HTML Help Workshop** (`hhc.exe`).

```bash
# 1. распаковать оригинал (7-Zip) во временную папку
7z x ../../help.CHM -o_chm_src

# 2. перевести (создаёт _chm_build/ с HTML, картинками, toc.hhc, index.hhk, help_ru.hhp)
python build_ru.py            # пути можно переопределить через CHM_SRC / CHM_OUT

# 3. перед компиляцией сравнить заменённые картинки с оригиналом
python compare_manual_screenshots.py --original _chm_src --candidate _chm_build --out _frame_compare

# 4. скомпилировать CHM (hhc.exe возвращает 1 даже при успехе — проверяйте сам файл)
"/c/Program Files (x86)/HTML Help Workshop/hhc.exe" _chm_build/help_ru.hhp

# 5. забрать результат
cp _chm_build/help_ru.chm ../../help_ru.chm

# 6. проверить видимый brand в CHM
python check_chm_brand.py ../../help_ru.chm --forbid ZTool --require SWTools
```

## Что делает `build_ru.py`

1. **ASCII-имена.** Китайские имена тем и папок (`功能介绍` → `features`,
   `基本操作` → `basic` …) переименовываются в ASCII — иначе пути внутри CHM
   ломаются.
2. **Перевод текстовых узлов.** HTML делится на теги/текст; переводятся только
   текстовые узлы (блоки `<style>/<script>/<xml>` и комментарии пропускаются).
   Замена «от длинного к короткому», поддержаны и entity-варианты (`&lt;`/`&gt;`).
3. **Расстановка пробелов.** В китайском между стилизованными `<span>`-сегментами
   пробелов нет; после перевода на стыках вставляется пробел (`space_glue`).
4. **Кодировки.** Темы переписываются в `charset=utf-8`; навигация (HHC/HHK)
   пишется в `cp1251` с `Font … ,204` (русская кодовая страница).
5. **Проект.** Генерируется `help_ru.hhp` с `Language=0x419`, `Binary Index=No`
   (иначе индекс отображается «кракозябрами»), `Title` на русском.

## Что осознанно не переведено

- **Растровые скриншоты** внутри тем — это снимки самого (китайского) интерфейса
  программы; перевести их без перерисовки нельзя.
- Имя файла **`初始化.exe`** оставлено как есть (это реальное имя запускаемого
  файла) с пометкой «(Инициализация)».
