# SolidWorks-макросы

## WriteRussianProps.vbs

Записывает в активную сборку SolidWorks и во все её компоненты пользовательские
свойства с русскими (ЕСКД) именами, которые совпадают с `<propname>` /
видимыми колонками в `ZTool.settings`: `Наименование`, `Обозначение`,
`Материал`, `Тип`, `Версия`, `Обработка поверхности`, `Масса`.

Важно: `<mappingname>` в `ZTool.settings` — это не имя свойства модели, а
Excel-anchor шаблона. Он остаётся legacy/CN (`零件名称`, `图号`, `材质` и т.д.),
чтобы экспорт писал в существующие именованные диапазоны `bom_шаблон.xlsx`.

Значения:
- `Обозначение` = имя файла без расширения (например `0614-P001`);
- `Материал` = ссылка SolidWorks `SW-Material@<файл>` (только детали);
- `Масса` = ссылка SolidWorks `SW-Mass@<файл>` (резолвится самим SW);
- `Тип` = `Мех.обработка` для деталей и `Сборка` для сборок — **временная
  заглушка**, чтобы фильтр-режимы спецификации классифицировали детали (значения
  задаются константами `T_PART` / `T_ASSEMBLY` в начале скрипта, поправьте под
  реальную классификацию позже);
- `Наименование`, `Версия`, `Обработка поверхности` создаются пустыми.

Свойства пишутся и на уровне документа, и на уровне активной конфигурации.

### Запуск

1. Открыть в SolidWorks сборку `0614-A00`, полностью разрешённую (не Lightweight
   и не Large Design Review), сделать активной.
2. Запустить: двойной клик по `WriteRussianProps.vbs` или `cscript WriteRussianProps.vbs`.
3. Сделать «Сохранить всё» (макрос сохраняет каждый документ сам, но повторное
   сохранение не повредит).

### Примечание про «Тип»

Фильтр-режимы спецификации (`FilterRulesList`: «Обрабатываемые детали»,
«Покупные изделия» и т.д.) опираются на свойство `Тип`. Скрипт ставит дефолт
`Мех.обработка` (детали) / `Сборка` (сборки) как временную заглушку — этого
достаточно, чтобы режим «Обрабатываемые детали» классифицировал детали.
Реальную классификацию (Покупное/Литьё/Сварка/…) задайте позже — в SW или
правкой констант `T_PART` / `T_ASSEMBLY` в скрипте.
