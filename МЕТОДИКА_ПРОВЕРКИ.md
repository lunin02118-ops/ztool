# Методика проверки и тестирования ZTool (интеграционная сборка PR #7)

Документ описывает пошаговую проверку и тестирование сборки `integration/all-prs`
(PR #7), которая объединяет все 6 открытых PR + русский BOM-шаблон + фикс
безопасности сервера лицензирования.

Цель — подтвердить, что **функциональность и реализация идентичны оригиналу**, а
русская локализация (включая BOM-шаблоны) настроена корректно.

## 2. Сервер лицензирования — автотесты

### 2.1. Установка зависимостей

```bash
cd license-server
python -m pip install setuptools wheel pytest pytest-asyncio pycryptodome
python -m pip install -e . --no-build-isolation
```

### 2.2. Запуск тестов

```bash
python -m pytest -q
```

**Ожидаемо:** `66 passed, 1 skipped`.

Ключевые тесты:
- `tests/test_protocol.py` — кадрирование TCP `[type:10][len:10][AES-Base64]`.
- `tests/test_crypto.py` — RSA-1024 / AES-128-CBC / DES, совместимость с `ZTool_rsa.dll`.
- `tests/test_integration.py` — полный цикл активации (apply 128 → blob → confirm 131).
- `tests/test_integration.py::test_protected_code_rejects_empty_password` —
  **регрессия фикса безопасности**: код с паролем не активируется при пустом/неверном пароле.
- `tests/test_machineid.py` — привязка к железу (валидация 36-символьного GUID).

### 2.3. Проверка фикса безопасности (вручную)

Подтвердить, что обход пароля закрыт:

```bash
python -m pytest tests/test_integration.py::test_protected_code_rejects_empty_password -v
```

**Ожидаемо:** `PASSED`. (Пустой пароль на защищённом коде → результат `WRONG_PASSWORD`, место лицензии не расходуется.)

## 4. Русский BOM-шаблон — проверка целостности и раскладки

Файл: `Шаблоны спецификации/bom_шаблон.xlsx` (на него ссылается `ZTool.settings`).

### 4.1. Автоматическая проверка (целостность + заголовки)

```bash
python - <<'PY'
import zipfile, glob, openpyxl
src=[p for p in glob.glob('BOM*/**',recursive=True) if p.lower().endswith('.xlsx')][0]
out='Шаблоны спецификации/bom_шаблон.xlsx'
zs, zo = zipfile.ZipFile(src), zipfile.ZipFile(out)
assert zo.testzip() is None, "битый zip"
assert set(zs.namelist())==set(zo.namelist()), "набор частей книги отличается"
diff=[n for n in zs.namelist() if n!='xl/sharedStrings.xml' and zs.read(n)!=zo.read(n)]

wb=openpyxl.load_workbook(out); ws=wb.active

print("Заголовки строки 6:", [ws.cell(6,c).value for c in range(1,16)])
print("OK: книга валидна, изменён только sharedStrings.xml")
PY
```

**Ожидаемо:**

- изменён **только** `xl/sharedStrings.xml` (остальные 12 частей байт-в-байт);

## 6. Пересборка и проверка клиента ZTool.exe

> Требуется .NET SDK. Каталог `client-core/` содержит редактируемое лицензионное
> ядро на C# + реинжект в оригинальный `ZTool.exe` (GUI/ресурсы не трогаются).

### 6.1. Сборка

```powershell
cd client-core
./build.ps1
```

Пайплайн: публицайз базового exe → компиляция `src/*.cs` → реинжект тел методов в
копию `ZTool.exe` → локализация оставшихся китайских строк форм → проверка отсутствия
висячих ссылок. Результат: `client-core/out/ZTool.exe`.

### 6.2. Проверка перевыпуска под наш сервер (client-rekey)

`client-rekey/` патчит в exe адрес/порт/ключ нашего сервера
(`<LICENSE_SERVER_IP>:58000`, версия `1.0`). Сверить значения:

```bash
cat client-rekey/new_ip.txt client-rekey/new_port.txt   # RSA-блобы, патчатся в exe
```

### 6.3. Регистрация аддона в SolidWorks

```powershell
# от администратора, путь к собранному exe/dll
RegAsm.exe /codebase ZTool.dll
```

**Критерий приёмки:** `build.ps1` завершается без ошибок, `out/ZTool.exe`
запускается, подключается к нашему серверу активации, GUI приложения на русском.

> ВАЖНО: `build.ps1` русифицирует **`ZTool.exe`**. Вкладка/панель ZTool внутри
> SolidWorks рисуется аддоном **`ZTool.dll`**, который в PR #7 **не переведён** —
> см. раздел 9 (известное ограничение P1).

## 9. Известные ограничения

- **`ZTool.dll` — вкладка ZTool в SolidWorks остаётся китайской (P1).**
  Аддон SolidWorks — это `ZTool.dll`, а не `ZTool.exe`. DLL — **защищённая
  (обфусцированная) сборка**: её 188 китайских UI-строк (零部件重命名 «Переименование
  деталей», 保存选中项, 随机上色, 文件管理, 批量打印, 关于 …) лежат в куче пользовательских
  строк (`#US`, 13320 байт), но **тела методов зашифрованы протектором** — инструмент
  локализации (dnlib), которым русифицирован `ZTool.exe`, видит во всей сборке лишь
  8 `ldstr` и 0 китайских (формы вроде `Frm_SN` без видимого `InitializeComponent`).
  Поэтому:
  - переписать строки через dnlib/Localizer нельзя (инструмент не видит `ldstr`,
    а перезапись повредит защищённую сборку);
  - побайтовый патч `#US` тоже не подходит: русский текст длиннее китайского и не
    помещается в исходные слоты, а сдвиг ломает offset-токены `ldstr`;
  - исходников UI-аддона в репозитории нет (в `client-core/src` только
    лицензионное ядро `SR/SecurityCenter/TCPClient/...`), поэтому пересобрать DLL
    из исходников невозможно.
  Возможные пути (требуют решения заказчика): (1) деобфускация/дамп DLL → русификация
  Localizer'ом → распространение без защиты (меняет бинарь, нужна проверка в SW);
  (2) получить незащищённую/исходную сборку аддона у правообладателя.
- **`.sldbomtbt`** (`SW模板/材料明细表.sldbomtbt`) — бинарный шаблон таблицы BOM
  внутри SolidWorks. Заголовки в нём **не русифицированы** (правка требует SW и
  не входит в этот PR). Если требуется русская таблица BOM непосредственно в
  чертеже SW — это отдельная задача.
- **E2E экспорт BOM** проверяется только в интерактивной сессии SolidWorks;
  автоматизировать в данном окружении нельзя (нет SW / COM-автозапуск зависал).
- **Сборка клиента** требует .NET SDK (`net48` таргет); в текущем окружении CI
  для C# не настроен.
- Абсолютный путь в `ZTool.settings` (`C:\Users\Administrator\wt_client\_runmax\…`)
  соответствует ожидаемому каталогу установки — как и в оригинале (оригинал тоже
  использовал абсолютные пути). При другой папке установки путь нужно поправить.
