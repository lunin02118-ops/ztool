# МЕТОДИКА ТЕСТИРОВАНИЯ ZTool — всё приложение (DLL + EXE + сервер лицензий)

Цель: единый прогон всех компонентов на твоей машине с SolidWorks, заполнение
шаблона отчёта (раздел 12) и возврат результата мне. По нему я итеративно
дочищаю оставшиеся дефекты.

ВАЖНО про язык скринов: окна с китайским триал-таймером `试用中...剩余N秒`,
статусом `连接完成/耗时/共` или попапом `Найдена новая версия` — это СТАРЫЙ `ZTool.exe`.
В правильном билде их нет. Сначала сверь хеши (раздел 1).

## 1. Подготовка окружения (обязательно)

1.1. Создай ASCII-папку без кириллицы/китайского в пути, напр. `D:\ztool-test\`.
     (Китайские/кириллические символы в пути ломают чтение моделей.)

1.2. Удали ВСЕ старые копии `ZTool.exe` и `ZTool.dll` из мест прошлых тестов
     (особенно из `...\prs\pr-7-8926c89\...` — там устаревшая сборка).

1.3. Положи в `D:\ztool-test\` три файла: `ZTool.dll`, `ZTool.exe`, `SolidWorksTools.dll`.

1.4. Сверь хеши (PowerShell):
```powershell
Get-FileHash D:\ztool-test\ZTool.dll  -Algorithm SHA256   # ожидаем eea7c9ae...ba0961
Get-FileHash D:\ztool-test\ZTool.exe  -Algorithm SHA256   # ожидаем 9397312e...d47de21
```
     Если хеш другой — это НЕ тот файл, тест дальше не имеет смысла.

1.5. Подготовь тестовую сборку с несколькими деталями и конфигурациями
     (как `0614-A00.SLDASM`). Желательно с BOM-таблицей.

1.6. Если есть оригинальный (китайский) `ZTool.exe` и оригинальная `ZTool.dll` —
     сохрани их отдельно для эталонного сравнения функций (раздел 7).

> Заметка: SolidWorks тестируется на твоей стороне — у меня COM SW недоступен.

## 2. Регистрация аддона

2.1. От администратора:
```cmd
cd /d D:\ztool-test
"C:\Windows\Microsoft.NET\Framework64\v4.0.30319\RegAsm.exe" ZTool.dll /codebase
```
     Ожидаем: `Types registered successfully`. Текст ошибки — в отчёт (3.1).

2.2. (Опц.) Проверка ключей реестра аддона:
```powershell
Get-ChildItem "HKLM:\SOFTWARE\SolidWorks\AddIns" -ErrorAction SilentlyContinue
Get-Item "HKCU:\Software\SolidWorks\AddInsStartup\{59959DFA-3229-4B86-852E-52ABF2BDB8C0}" -ErrorAction SilentlyContinue
```

## 5. ZTool.exe: запуск и язык

Кнопки-лаунчеры ленты (Управление файлами, Пакетное преобразование форматов,
Пакетная печать, Объединение и разделение PDF, Замена рамки и стандарта,
Замена ссылочных файлов, Синхронизация имён чертежей, О программе) запускают `ZTool.exe`.

Проверь в окне ZTool.exe:
5.1. **Заголовок окна / триал-таймер** — на русском («Пробный период... осталось N сек»), НЕ `试用中...剩余N秒`.
5.2. **Статус-бар внизу** — на русском (НЕ `连接完成/耗时/共`).
5.3. Вкладки/меню (Файл, Главная, Спецификация, Упаковка, Инструменты) — русские?
5.4. Заголовки таблиц (Имя файла, Конфигурация детали, Материал, …) — русские?
5.5. **Обновления:** при старте exe попап «Найдена новая версия» НЕ всплывает;
     кнопка «Обновление» открывает окно с текстом «Обновления отключены»; сети к `z-tool.cn` нет.
5.6. **Диалог ошибок** (если выскочит) — «Тип исключения / Сообщение / Информация / Стек», не китайский.
5.7. **О программе** — русское.
5.8. Любой оставшийся китайский — скриншот окна целиком в отчёт.

## 6. ZTool.exe: ФУНКЦИИ (паритет с оригиналом)

Базовый прогон ключевых функций. Для каждой: работает / ошибка (+текст).

6.1. **Чтение файлов из SW** (приоритет — был баг «0 поз.»):
   1) Открой сборку `0614-A00` в SW.
   2) Лента ZTool → «Управление файлами» (откроется ZTool.exe).
   3) Кнопка **«Подключить SW»**.
   4) Режим чтения: проверь ОБА — «Чтение по спецификации» и «Читать всё».
   5) Сколько позиций прочитано? (ожидаем > 0; в отчёт — число и режим)
6.2. Переименование деталей (через exe, если применимо) — ок?
6.3. BOM / экспорт спецификации — ок?
6.4. Пакетная печать — ок?
6.5. Объединение и разделение PDF — ок?
6.6. Пакетное преобразование форматов — ок?

## 7. ЭТАЛОННОЕ СРАВНЕНИЕ с оригиналом (для локализации регрессий)

Только если функция в моём билде НЕ работает. Тот же шаг на ОРИГИНАЛЬНОМ
(китайском) `ZTool.exe` + оригинальной `ZTool.dll`:

7.1. Чтение по спецификации в оригинале даёт позиции или тоже 0?
   - Если в оригинале РАБОТАЕТ, а в моём нет → это моя регрессия (чиню).
   - Если в оригинале ТОЖЕ 0 → дело в окружении/порядке действий, не в локализации.
7.2. Любая другая «сломанная» функция — так же сравни с оригиналом и отметь.

## 8. Сервер лицензий (P1 / P2) — опционально, без SW

8.1. Установка (дефект P1):
```bash
cd license-server
pip install -e .
```
     Ожидаем: установка без ошибок (раньше падала).

8.2. Переменные окружения (дефект P2): сервер должен читать `ZTOOL_*`:
```bash
set ZTOOL_HOST=127.0.0.1
set ZTOOL_PORT=8080
python -m ztool_license_server   # или точка входа из README сервера
```
     Ожидаем: слушает на заданных host/port из env.

8.3. Тесты:
```bash
cd license-server
pytest -q
```
     Ожидаем: все тесты зелёные.
