# Шаблоны спецификации (BOM)

Русские шаблоны для экспорта спецификаций SWTools. `bom_шаблон.xlsx` — это
рабочий шаблон с русскими видимыми заголовками и русскими Excel anchors.
Заголовки в строке 6 и defined names должны оставаться синхронными с
`mappingname` в настройках, иначе экспорт не найдёт куда писать данные.

Дополнительно в русской поставке есть русские Excel anchors для расчётных
колонок SWTools:

| Колонка SWTools | Anchor в шаблоне | Ячейка |
|---|---|---|
| `Масса` (`Col_Weight`) | `Масса` | `J6` |
| `Габарит` (`Col_bound`) | `Габарит` | `P6` |
| `Ед. изм.` | `ЕдИзм` | `F6` |

Эти anchors проверяются `client-core/tools/check_bom_template.py`; удалять или
переименовывать их нельзя без синхронного изменения кода экспорта.

## Важно: путь к шаблону должен быть АБСОЛЮТНЫМ

Клиент открывает шаблон по пути из `<bomname>` напрямую. Относительный путь
.NET разрешает относительно рабочей папки процесса, а под SolidWorks это
`…\Documents` — поэтому относительный путь приводит к ошибке «не удалось найти
файл». В оригинале путь был абсолютным.

По умолчанию в `SWTools.settings` прописано:

```
D:\SWTools\Шаблоны спецификации\bom_шаблон.xlsx
```

Если SWTools установлен в другую папку — перенацельте путь одной командой
(перепишет все `<bomname>` и `<bompath>`):

```powershell
client-core\tools\set_bom_template_path.ps1 -Folder "D:\ваш\путь" -Settings "D:\ваш\путь\SWTools.settings"
```

(в `-Folder` указывается папка, содержащая подпапку `Шаблоны спецификации`).

## Свойства модели

Колонки данных заполняются из **пользовательских свойств SolidWorks**.
Имена свойств берутся из `<propname>` в `SWTools.settings` и должны совпадать с
именами свойств в ваших моделях (у вас они русские: «Имя детали»,
«Номер чертежа», «Материал», «Версия», «Обработка поверхности», «Масса» и т.д.).
Если в модели свойство названо иначе — соответствующая колонка будет пустой.

## Типы отчётов (7 пресетов)

| Пресет | type | Эскизы | Фильтр |
|---|---|---|---|
| Сводная спецификация | 0 | нет | по фильтру |
| Сводная спецификация (с эскизами) | 0 | да | по фильтру |
| Иерархическая спецификация | 1 | нет | по фильтру |
| Иерархическая спецификация (с эскизами) | 1 | да | по фильтру |
| Покупные детали | 0 | нет | по правилу `Покупные` |
| Внешние детали | 0 | нет | по правилу `Внешние` |
| Спецификация верхнего уровня | 2 | нет | по правилу |

Пресеты «Покупные детали» / «Внешние детали» фильтруют строки по правилу
(`RulesList`). Значения `Покупные` / `Внешние` — заготовки: замените их на тот
признак классификации, который реально используется в ваших моделях.
