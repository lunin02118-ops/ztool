# Методика тестирования генерации отчётов BOM

Документ описывает, как проверить, что экспорт спецификаций (BOM) из ZTool
работает корректно для всех типов отчётов. Экспорт физически выполняется
**внутри SolidWorks** (ZTool — аддон, который читает дерево сборки через API
SolidWorks), поэтому полноценная проверка возможна только на машине с
установленным SolidWorks и реальной русской моделью.

---

## 0. Предусловия

- Установлен SolidWorks, ZTool-аддон подключён, лицензия активна.
- Открыта **русская** сборка (`.SLDASM`), у компонентов заданы пользовательские
  свойства с именами из таблицы ниже (Имя детали, Номер чертежа, Материал …).
- `ZTool.settings` и папка `Шаблоны спецификации` доступны по абсолютному пути.

---

## 1. Pre-flight (офлайн, без SolidWorks)

Этот шаг ловит ошибки конфигурации (неверный путь к шаблону, отсутствующие
именованные диапазоны) **до** запуска SolidWorks:

```
python client-core\tools\check_bom_template.py "D:\ваш\путь\ZTool.settings"
```

Ожидаемо: `RESULT: PASS`. Проверяет:
- путь к шаблону у каждого пресета — абсолютный;
- каждый `<mappingname>` из `namemappinglist` существует как именованный
  диапазон в `bom_шаблон.xlsx`;
- список из 6 типов отчётов.

Если `FAIL` — устранить до перехода к SolidWorks.

---

## 2. Перенацелить путь шаблона на свою папку установки

Если ZTool установлен не в `D:\ZTool`:

```powershell
client-core\tools\set_bom_template_path.ps1 -Folder "D:\ваш\путь" -Settings "D:\ваш\путь\ZTool.settings"
```

Скрипт перепишет `<bompath>` и `<bomname>` во всех пресетах. Перезапустить ZTool,
чтобы он перечитал настройки.

---

## 3. Соответствие колонок (приёмочная таблица)

После экспорта на листе `图明1` шаблона данные раскладываются так (строка
данных начинается с **строки 6**, далее вниз по числу компонентов):

| Свойство модели (propname) | Колонка PropVal | Анкер шаблона | Ячейка |
| --- | --- | --- | --- |
| Имя детали | PropVal_1 | `零名` | **C6** |
| Номер чертежа | PropVal_2 | `图` | **D6** |
| Версия | PropVal_5 | `版` | **E6** |
| Количество | PropVal_6 | `统数` | **G6** |
| Тип | PropVal_4 | `类` | **H6** |
| Обработка поверхности | PropVal_7 | `表处` | **I6** |
| Масса | PropVal_9 | `重` | **J6** |
| Материал | PropVal_3 | `材` | **N6** |
| № п/п (авто) | — | `序` | **A6** |
| Имя файла (авто) | — | `磁文名` | **L6** |
| Путь (авто) | — | `路` | **O6** |
| Эскиз (для пресетов «с эскизами») | — | `缩图` | **M6** |

> Нюанс: «Разработал» (PropVal_0) и «Дата разработки» (PropVal_8) в текущем
> шаблоне **не имеют** ячейки (нет именованного диапазона), поэтому в лист не
> выводятся. Если они нужны — добавьте в `bom_шаблон.xlsx` именованные
> диапазоны и пропишите их в `<mappingname>` соответствующих колонок.

---

## 4. Прогон 6 типов отчётов

Для каждого пресета: меню ZTool → выбрать пресет → экспорт → сохранить → открыть
полученный `.xlsx`/`.xls` и сверить с критериями.

| # | Пресет | type | Эскизы | Что ожидать |
| --- | --- | --- | --- | --- |
| 1 | Экспорт сводной спецификации | 0 | нет | Плоский список всех компонентов; одинаковые детали свёрнуты, в `统数` (G6…) — суммарное количество. |
| 2 | Экспорт иерархической спецификации | 1 | нет | Состав по уровням сборки (узлы и их детали), отражена вложенность. |
| 3 | Экспорт спецификации верхнего уровня | 2 | нет | Только компоненты верхнего уровня сборки (без раскрытия подсборок). |
| 4 | Экспорт сводной спецификации деталей | 3 | нет | Сводный список только деталей (без узлов/подсборок). |
| 5 | Экспорт сводной спецификации (с эскизами) | 0 | да | Как №1 + в колонке `缩图` (M6…) вставлены эскизы компонентов. |
| 6 | Экспорт иерархической спецификации (с эскизами) | 1 | да | Как №2 + эскизы в `缩图` (M6…). |

---

## 5. Критерии приёмки (PASS/FAIL)

PASS, если для каждого пресета:
- файл создаётся без ошибки «не удалось найти файл …\Documents\…» (значит путь
  абсолютный отработал);
- заполнена **не только шапка**, а строки данных с 6-й вниз (C6, D6, N6, G6 …)
  по числу компонентов модели;
- значения соответствуют свойствам модели (имя/номер/материал/кол-во и т.д.);
- для пресетов «с эскизами» — в `缩图` (M-колонка) присутствуют картинки;
- иерархия/верхний уровень/только-детали — соответствуют типу (см. таблицу §4).

FAIL фиксировать с деталями: имя модели, какой пресет, скрин Excel, какие
ячейки пусты, текст ошибки (если была).

---

## 6. Что приложить к отчёту

- Имя тестовой модели и число компонентов.
- По каждому из 6 пресетов: полученный файл (или скрин листа `图明1`).
- Любые пустые/неверные колонки — со ссылкой на строку таблицы §3.
